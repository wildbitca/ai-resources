# ClickUp Feature Tracking

- **REQUIRED:** Every new feature or request MUST belong to a ClickUp card—either an existing task or one created for the work.
- **REQUIRED:** Before starting feature work: search ClickUp for an existing card; if none exists, create a task in the correct list (e.g. Mobile / Backend / OPS) following project patterns (epic-style organization, naming, status, tags).
- **REQUIRED:** Use ClickUp MCP tools: `clickup_search`, `clickup_get_workspace_hierarchy`, `clickup_create_task`, `clickup_get_list` (for list IDs).
- **BENEFIT:** Traceability, consistent epic/list/status/tag organization, single source of truth.
